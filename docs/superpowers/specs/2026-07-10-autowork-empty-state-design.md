# AutoWork 标签选择空状态设计

## 背景

会话和终端顶部的“自动工作”弹层通过需求标签决定要处理的需求队列。当需求平台尚未创建任何需求时，标签选择器只显示通用的“暂无数据”，既没有解释标签从何而来，也没有提供下一步操作。

本次优化把这个死胡同改为可行动的空状态，引导用户直接创建第一条需求，同时保持需求创建逻辑只存在于需求平台。

## 目标

- 没有需求标签时，明确说明标签由带标签的需求产生。
- 提供单一、明确的主操作，直达需求平台的新建需求抽屉。
- 区分加载中、空数据和加载失败，避免把短暂加载或网络错误显示成真实空状态。
- 没有可选标签时阻止开启自动工作，避免先允许操作再用警告纠错。
- 保持有标签时的现有选择、状态展示和启停行为不变。
- 同时支持简体中文和英文，并沿用现有主题令牌与组件体系。

## 非目标

- 不在 AutoWork 弹层内复制或嵌入需求表单。
- 不允许创建没有需求的孤立标签。
- 不改变需求、标签或 AutoWork 的后端接口和数据模型。
- 不新增创建完成后自动返回原会话的跨页面流程。

## 交互设计

### 有标签

标签选择器继续显示 `标签名 (已完成数/总数)`。AutoWork 已开启时选择器仍不可编辑，停用后才可更换标签。

### 加载中

首次请求标签时，下拉内容显示紧凑的加载指示与“正在加载标签”文案。此时不显示空状态，AutoWork 开关不可开启。

### 空数据

标签请求成功且结果为空时，下拉内容使用紧凑的行动型空状态：

- 图标：小尺寸 `ListAdd` 线性图标，放在柔和主题色底的圆形容器中。
- 标题：`还没有可用标签`。
- 说明：`标签会在创建需求后自动生成。先去需求平台添加一条带标签的需求。`
- 主操作：`新建需求`。

按钮跳转到 `/requirements?new=1`，复用需求平台现有的新建需求抽屉。路由切换会自然卸载当前弹层，不额外维护嵌套弹层状态。

没有标签时 AutoWork 开关处于禁用状态。选择器本身保持可点击，以便用户发现和使用空状态中的主操作。

### 加载失败

标签请求失败时，下拉内容显示：

- 标题：`标签加载失败`。
- 说明：`请检查连接后重试。`
- 次操作：`重新加载`，调用标签查询的 `refresh` 方法。

加载失败不得退化为空数据文案，也不得跳转到新建需求。

## 视觉规范

空状态适配当前约 240px 宽的 AutoWork 面板，不使用面向整页场景的大尺寸默认 Empty 组件：

- 内容居中，使用紧凑垂直间距与适量内边距。
- 标题使用 13px 中等字重和主文本色。
- 说明使用 12px、16px 行高和三级文本色，最多形成数行短文案。
- 按钮使用小尺寸主按钮；颜色、边框、背景全部使用现有主题令牌。
- 图标只辅助识别，不承担唯一的信息表达。

该层级让主操作清晰可见，但不会使下拉空状态在整个 AutoWork 弹层中喧宾夺主。

## 组件与数据流

### `useRequirementTags`

该 hook 在保留 `tags` 与 `refresh` 的基础上返回：

- `loading: boolean`：当前是否正在请求标签。
- `error: string | null`：最近一次未被全局鉴权流程处理的加载错误。

每次刷新开始时设置 `loading=true`；成功时更新标签并清空错误；失败时保留当前标签数据并记录错误；结束时设置 `loading=false`。已有需求实时事件仍触发刷新。

### `AutoWorkControl`

组件通过 `useNavigate` 执行新建需求跳转，根据 hook 状态为 Select 提供定制 `notFoundContent`：

1. `loading` 优先显示加载状态。
2. `error` 且没有可展示标签时显示失败与重试状态。
3. 请求成功且 `tags.length === 0` 时显示行动型空状态。
4. 有标签时渲染现有选项列表。

AutoWork 开关的禁用条件为：当前正在加载、加载失败且没有标签，或标签列表为空。已有已启用绑定不因一次刷新失败而被强制关闭；只有尝试从关闭状态开启时才应用该禁用条件。

## 可访问性与事件处理

- 主按钮与重试按钮使用原生 Button 语义，可通过 Tab 聚焦并用 Enter/Space 激活。
- 图标标记为装饰性，不重复朗读。
- 按钮文案独立表达目的，不依赖周围说明。
- 点击下拉内按钮时必须可靠触发操作，不把事件误当成 Select 选项选择。
- 加载指示提供可读文本，不能只依赖旋转图形。

## 国际化

在 `requirements.autowork` 下新增中英文键：

- `emptyTitle`
- `emptyDescription`
- `emptyCta`
- `loadingTags`
- `loadErrorTitle`
- `loadErrorDescription`
- `retry`

修改后重新生成并校验 `i18n-keys.d.ts`，不在 JSX 中使用只覆盖单语言的硬编码兜底文案。

## 测试与验收

先增加会失败的回归测试，再实现功能。测试至少验证：

- AutoWork 标签选择器接入自定义 `notFoundContent`。
- 空状态包含解释文案和新建需求主操作。
- 主操作指向 `/requirements?new=1`。
- hook 暴露加载与失败状态，成功刷新会清除旧错误。
- 无可用标签时不能开启 AutoWork；有标签时保留原行为。
- 中英文新增键一致，生成的类型声明保持同步。
