# 需求看板底部进度轨道设计

## 目标

优化需求平台的看板视图：看板列高度随卡片内容自然扩展；在应用视口最底部提供一条常驻、通栏的浅色分段进度轨道。鼠标悬停或键盘焦点进入某列时，仅对应分段以该状态的主题色加深并小幅增高。轨道的悬浮变化不得推动或重新排布看板内容。

## 范围

仅修改 `RequirementBoardView` 及其配套样式/测试契约。不改变需求数据、状态流转、拖放、横向滚动、卡片结构或其他页面的布局。

## 结构与行为

- 看板外层为相对定位的横向滚动容器，保留六列的既有顺序和最小宽度。
- 每一列移除现有的固定最大高度和内部纵向滚动，让列与看板容器由卡片内容撑开。
- 在同一组件中渲染一个 `aria-hidden` 的进度轨道；它以 `position: fixed` 覆盖应用视口最底缘，不参与文档流，也不增加看板高度或改变页面滚动尺寸。
- 固定轨道占满视口宽度，并以六个等宽分段表达状态列；横向滚动不移动轨道。
- 轨道由六个等宽分段组成，按列状态对应。默认使用低对比度、主题安全的填充色；不表达完成比例，不绑定卡片数量。
- 列的 `pointerenter` / `pointerleave` 与 `focusin` / `focusout` 共同维护当前活动状态。只有活动列的对应分段提高不透明度、使用该状态强调色，并向上扩展少量高度。拖拽放置的高亮保持现有语义，可同时让对应轨道分段处于强调态。
- 对低动效偏好，取消轨道和列的过渡效果。

## 主题与无障碍

- 颜色只来自 Arco/项目主题变量与已有各状态 accent；不写死亮色或暗色背景。
- 默认轨道在浅色、暗色和自定义主题中保持低调但可见；活动态使用状态色并保留足够区分度。
- 键盘进入卡片时，与该卡片所属列相应的轨道分段也会激活。轨道仅为视觉辅助，因此不新增焦点停靠点或辅助技术冗余信息。

## 验收与测试

- 源码契约测试验证：六个状态均产生对应的轨道分段，列高度不再受 `max-h-[calc(100vh-260px)]` 限制，轨道采用固定定位，活动态和 reduced-motion 规则存在。
- 运行 UI 类型检查与相关 Bun 测试。
- 人工检查有卡片、空列、横向溢出、拖放、鼠标悬停、键盘焦点，以及明暗主题下的显示；确认轨道动画未改变页面布局。
