# 桌面伙伴记忆气泡自适应布局设计

## 目标

修复桌面伙伴角标所打开的记忆气泡遮挡伙伴、正文被截断及长内容不可完整浏览的问题，同时保证透明原生窗口在屏幕边缘、不同缩放比例、自定义大尺寸伙伴和多显示器环境下不会裁切、跳屏或产生位置漂移。

本设计中的“记忆气泡”沿用用户界面称呼，对应现有代码中的待处理建议面板 `nomi-companion-suggestions`；不改变建议的来源、数量、点击跳转或已读语义。

## 根因

当前建议面板以绝对定位挂在伙伴立绘舞台内部，固定使用 `top: 32px`，因此面板天然占用立绘所在区域并覆盖伙伴。面板正文还使用两行截断，导致内容无法完整阅读。打开建议面板没有参与原生 Tauri 窗口的扩展状态，单纯把 CSS 面板向上移动会越过 WebView 边界并被透明窗口裁切。

## 方案选择

### 采用：自适应独立面板

将记忆气泡从立绘舞台中移出，作为伙伴旁边的独立布局区域。打开面板时，根据当前显示器的可用空间扩展原生窗口并选择布局方向；关闭时恢复打开前的伙伴窗口位置和尺寸。正文完整换行，面板达到高度上限后仅内容区滚动。

这是唯一同时解决角色遮挡、原生窗口裁切、长内容阅读和边缘环境稳定性的方案。

### 不采用：仅调整绝对定位

把当前面板改为负 `top` 或更大的偏移量改动较小，但内容会越过原生 WebView 边界，在屏幕顶部和小尺寸窗口中仍然裁切。

### 不采用：固定在伙伴单侧

固定左侧或右侧可以避开立绘，但伙伴靠近对应屏幕边缘时仍会越界，而且不同显示器与自定义尺寸下无法保持稳定。

## 布局与视觉

记忆气泡是独立的 `memory-panel` 区域，立绘舞台只保留角标和角色。布局优先级如下：

1. 当前显示器上方空间至少容纳“面板期望高度 + 12px 间距”时，面板位于伙伴上方、水平居中。
2. 上方不足时，在左、右空间中选择可用宽度更大的一侧；宽度相同时优先右侧。
3. 上、左、右都放不下期望尺寸时，选择可用面积最大的方向，窗口钳制在当前显示器工作区内并缩小面板可用尺寸；正文仍通过内部滚动完整可达。

面板与伙伴保持 12px 安全间距，不允许视觉包围盒相交。面板期望宽度为 `clamp(300px, 32vw, 360px)`，并受当前显示器可用宽度限制。内容区最大高度为 `min(320px, 42vh)`，期望最小可读高度为 160px；只有所有候选方向都不足 160px 时才使用最大实际可用高度。面板自身不会随内容无限增长。

视觉继续复用现有主题变量、圆角和阴影语言：14px 圆角、1px 主题边框、基础背景和克制阴影。条目之间使用 8px 间距和现有悬停底色，不增加高对比分割线；标题保持 12px/700，正文使用 12px、1.55 行高。移除正文两行截断，允许自然换行；长链接、长英文和代码使用 `overflow-wrap: anywhere`，不得撑破面板。

内容区使用细滚动条和 `overscroll-behavior: contain`，避免滚动传递到底层桌面。仅在确实溢出时出现滚动能力；顶部或底部仍有内容时显示轻微渐隐提示。

## 原生窗口与位置恢复

角标打开面板前，记录当前桌面态窗口的物理像素矩形、所在显示器和缩放比例，作为一次展开会话的不可变锚点。内部布局引起的 `setSize`、`setPosition` 不得写入伙伴配置中的用户位置。

窗口扩展使用当前显示器的物理坐标计算，在逻辑尺寸与物理尺寸之间统一通过 `scaleFactor` 转换。优先保持伙伴在屏幕上的视觉位置不变；需要翻转到侧边时只改变透明窗口包围盒和面板方向。关闭面板后精确恢复打开前的桌面态尺寸与位置，不使用扩展后的位置重新推导，防止反复开关后伙伴漂移。

多显示器场景始终以伙伴展开前所在显示器为宿主，包括带负坐标的左侧或上方显示器。不得回退到主显示器，除非原显示器在展开期间已被移除；此时才选择与旧矩形重叠最多的现存显示器并钳制到其工作区。

## 状态与交互

- 角标改为语义化按钮，暴露 `aria-expanded` 和面板关联关系；点击保持现有开关行为。
- 面板打开后继续纳入 `data-companion-hit` 与整窗捕获策略，滚动、条目点击和键盘操作不会穿透到底层窗口。
- 点击条目仍执行现有跳转，不在气泡内自动接受或忽略建议。
- `Escape` 关闭面板并恢复桌面态窗口；面板关闭后焦点回到角标按钮。
- 记忆气泡、对话气泡和展开输入框共享同一套窗口展开会话。记忆气泡打开时进入专注阅读态：对话气泡和输入框仅暂时隐藏、内部状态继续保留；关闭记忆气泡后恢复原状态，避免多个表面争抢高度或互相覆盖。
- 建议列表为空或在打开期间被外部处理完时，立即关闭面板并恢复窗口，不保留空白浮层。

## 组件边界

新增纯布局计算模块，输入为伙伴锚点矩形、面板期望尺寸、显示器工作区和缩放比例，输出为窗口矩形、面板方向及面板可用高度。它不访问 DOM、不调用 Tauri，也不读写配置，以便覆盖所有边缘条件。

React 页面负责测量实际内容、管理面板开关和调用布局计算；原生窗口适配层只执行尺寸与位置变化，并区分“内部布局移动”和“用户拖动”。CSS 只负责已选方向下的排版、视觉和滚动，不再承担屏幕空间判断。

## 异常与降级

无法读取显示器或缩放信息时，使用当前窗口所在坐标和 `devicePixelRatio` 计算上方布局，并把窗口钳制到可获得的视口边界。原生窗口扩展失败时，保持伙伴可见，面板使用当前窗口内的最大可用高度滚动，同时记录错误；不得显示越界或不可关闭的面板。

窗口恢复失败时保留当前可见位置，并允许下一次关闭或桌面尺寸同步重试；不得清空用户持久化位置。

## 测试与验收

### 自动化回归测试

- 伙伴位于屏幕中央时选择上方布局，面板与立绘间距至少 12px。
- 伙伴靠近顶部时自动选择空间更充足的左侧或右侧。
- 伙伴位于左、右、上、下边缘时，窗口和面板均不超出宿主显示器工作区。
- 125%、150%、200% 缩放下，逻辑尺寸和物理坐标转换一致。
- 自定义伙伴高度达到现有上限时，面板仍不与立绘相交。
- 多显示器包含负坐标、不同分辨率和不同缩放比例时，面板留在伙伴所在显示器。
- 超长正文、无空格长字符串和多条建议完整可达，达到高度上限后内容区滚动。
- 连续打开、关闭面板后，桌面态窗口精确恢复原矩形，且内部移动不写入伙伴配置。
- 建议在打开期间清空时，面板关闭并恢复窗口。

### 视觉验收

- 以用户截图对应状态验证：气泡完全位于伙伴安全区之外，角色头部与身体无遮挡。
- 每条标题与正文不出现省略号或两行截断；滚动到底部可读取最后一条完整内容。
- 浅色与深色主题下边框、背景、文字对比和滚动条均清晰。
- 面板打开、换向和关闭过程中无明显跳动、闪烁或伙伴位置漂移。

## 范围

本次只修复桌面伙伴角标面板的布局、正文呈现、滚动、窗口适配和可访问交互。建议分页、数据接口、业务状态、点击目的地、学习逻辑和主窗口中的建议列表保持不变。

## 设计自检

规范没有未决项或占位描述。布局方向、尺寸限制、位置恢复、缩放换算、异常降级和验收条件均有唯一解释；范围集中在记忆气泡显示问题，不包含无关重构。
