# ~help 命令 - 显示帮助

本模块定义帮助命令的输出内容。

---

## 命令说明

```yaml
命令: ~help
类型: 直接执行类
功能: 显示帮助信息，无需确认或评估，直接输出帮助内容
触发: 用户输入包含 ~help 或 ~?
```

---

## 执行模式适配

> 📌 规则引用: 按 G4 路由架构及 G5 执行模式规则执行

<mode_adaptation>
~help 模式适配规则:
1. 本命令为直接执行类，无需用户确认
2. 不受任何工作流模式影响
3. 检测上下文后直接输出帮助内容
4. 输出后保持待命状态，无状态变量设置
</mode_adaptation>

---

## 执行流程

### 步骤1: 上下文检测

<context_analysis>
上下文检测推理过程:
1. 检测用户输入是否携带上下文（疑问、困惑、请求等）
2. 如有上下文，准备简短回应用户的问题或情绪
3. 根据上下文调整下一步建议的针对性
</context_analysis>

```yaml
执行内容:
  - 检测用户输入是否携带上下文（疑问、困惑、请求等）
  - 如有上下文，准备简短回应用户的问题或情绪
  - 根据上下文调整下一步建议的针对性

目标: 友好引导，帮助用户快速上手
```

### 步骤2: 输出帮助信息

```yaml
执行内容:
  - 如步骤1检测到上下文，先简短回应
  - 按下方"输出模板"输出帮助内容
  - 按 G3 场景内容规则（直接回答）格式化输出

说明: 此命令为原子操作，无状态变量设置，输出后保持待命状态
```

### 可用命令

| 命令 | 功能 | 说明 |
|------|------|------|
| `~auto` | 全授权命令 | 根据复杂度自动选择模式执行 |
| `~plan` | 执行到方案设计 | 仅轻量迭代/标准开发模式 |
| `~exec` | 执行方案包 | 执行已有的方案包 |
| `~init` | 初始化知识库 | 扫描代码库创建知识库 |
| `~upgrade` | 升级知识库 | 升级知识库结构 |
| `~clean` | 清理遗留方案包 | 清理未执行的方案包 |
| `~commit` | Git 提交 | 智能生成提交信息并执行提交 |
| `~test` | 运行测试 | 运行测试并分析结果 |
| `~review` | 代码审查 | AI辅助代码审查 |
| `~validate` | 验证知识库 | 验证知识库和方案包 |
| `~rollback` | 智能回滚 | 回滚最近的变更 |
| `~help` | 显示帮助 | 显示此帮助信息 |

### 工作流程

需求评估 → 复杂度判定 → 对应模式执行
- 微调模式: 直接执行 → 知识库(KB)同步
- 轻量迭代: 项目分析 → 方案设计 → 开发实施 → 方案包归档
- 标准开发: 项目分析 → 方案设计（含多方案对比）→ 开发实施 → 方案包归档

### 使用方式

- 输入 `~命令` 执行特定功能
- 直接描述需求，系统自动路由到合适的流程
