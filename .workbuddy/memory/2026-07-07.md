## 修复 Canvas 节点数据传递
**背景**：检查 `wiki/flowgram-demo-and-doc` 文档后，发现 canvas 页面的节点数据传递存在 4 个 bug。

**发现的 Bug**：
1. `CanvasEditor.tsx:171` `referenceImages={[]}` 硬编码，已连接节点的图片数据从未传入 edit panel
2. `useEditorProps.tsx` 的 `onNodeClick` 接收了但从未挂接到 Flowgram 事件
3. edit panel 数据变更后只存 React state，不写回 Flowgram 节点
4. `ConnectedImages.ts` 的 `getConnectedTexts()`/`getConnectedImages()` 从没被调用

**修复内容**：
- **FlowEditor.tsx**：重构为内部 `FlowEditorInner`（用 `usePlaygroundContext()` 监听 selection 变化，计算 connectedImages/connectedTexts，通过 `onNodeSelect` 回传给父组件），新增 `NodeSelectPayload` 接口和 `playgroundRef` 暴露
- **useEditorProps.tsx**：移除无效的 `onNodeClick` 参数
- **CanvasEditor.tsx**：移除 `useNodeSelection` hook，改用 Flowgram 驱动的 selection；`handleNodeSelect` 接收完整 NodeSelectPayload；`referenceImages` 改为从 connectedImages 计算；`handleTextDataChange`/`handleImageDataChange` 新增同步写回 Flowgram node 的逻辑；新增取消选中清空 Flowgram selection
- **ConnectedImages.ts**：修复 `getData()` 类型问题（FlowGram API 需 Registry 参数，用 `(n as any).getData()` 绕过）
